# Pokdeng Online — 모바일 앱 구조 + 2재화(칩/다이아) 멀티플레이어

Node.js + Express + Socket.io 기반 실시간 포크덩(Pokdeng) 서버입니다.
모바일 게임 앱과 비슷한 구조(상단 재화 표시 + 하단 탭바: 로비/마켓)를 갖췄고,
칩(베팅 재화)과 다이아(구매 재화) 2개의 재화 시스템으로 동작합니다.

## 재화 구조

| 재화 | 용도 | 획득 방법 |
|---|---|---|
| 🪙 칩 (Chip) | 테이블 베팅 전용, 현금화 불가 | 게임에서 따거나, 다이아로 교환 |
| 💎 다이아 (Diamond) | 프리미엄 재화 | 앱스토어/구글플레이 실제 결제(IAP), 데모에서는 mock 구매 |

칩은 다이아로 교환해서 충전할 수 있지만, **칩→다이아 역교환이나 현금화는 없습니다.**
(현금화 가능한 실제 도박이 되지 않도록 하는 구조입니다.)

## 로비 — 배팅금액별 6단계 + 티어별 다중 방

각 티어(단계)는 방을 여러 개 가질 수 있어요. 방이 다 차면(6명) 자동으로 새 방이 열리고,
빈 방이 여러 개 남으면 자동으로 정리돼서 항상 딱 1개의 빈 방만 유지돼요.

| 단계 | 이름 | 최소 베팅 |
|---|---|---|
| 1 | 연습 테이블 | 10 칩 |
| 2 | 브론즈 테이블 | 100 칩 |
| 3 | 실버 테이블 | 500 칩 |
| 4 | 골드 테이블 | 2,000 칩 |
| 5 | 플래티넘 테이블 | 10,000 칩 |
| 6 | VIP 다이아 테이블 | 50,000 칩 |

### 입장 방식 2가지

- **⚡ 빠른 매칭**: 티어를 고르면 그 단계에서 사람이 가장 많이 모여있는(=가장 빨리 시작될) 방에 즉시 배정. 적합한 방이 없으면 새 방을 만들어서 입장.
- **🚪 직접 선택**: 티어 상세 화면에서 방 목록(방 #1, #2...)을 보고 원하는 방을 클릭해서 입장.

## 멀티플레이어 필수 요소 — 재접속(Reconnect) 지원

브라우저에 저장되는 영구 식별자(`clientId`, `localStorage`)를 기준으로 지갑과 좌석을 관리해서,
**새로고침하거나 네트워크가 잠깐 끊겨도 30초 안에 재접속하면 하던 방/좌석으로 자동 복귀**돼요.
(이전 버전은 `socket.id` 기준이라 새로고침하면 완전히 초기화됐는데, 이번에 개선했어요.)

- 방이 진행 중(betting/turns)일 때 끊겨도, 유예시간 안에 재접속하면 손에 든 카드/베팅이 그대로 유지돼요.
- 로비 단계에서 유예시간 안에 재접속 안 하면 방에서 자동으로 제외돼요.


## 마켓 — 재화 구매 및 교환

- **다이아 구매**: 6종 패키지 (일부는 보너스 다이아 포함) — `DIAMOND_PACKAGES`에서 가격/수량 조정
- **다이아 → 칩 교환**: 4종 환율 — `EXCHANGE_RATES`에서 조정
- 상품 정의는 `GET /api/market`으로 클라이언트에 내려줍니다.

### ⚠️ 실제 IAP(인앱결제) 연동에 대해

지금 "구매" 버튼은 **즉시 다이아를 지급하는 mock(테스트용) 로직**입니다.
실제 앱스토어/구글플레이 결제는 브라우저에서는 처리할 수 없고, **네이티브 앱**(React Native + `react-native-iap` 등)으로
패키징한 뒤 다음 흐름으로 교체해야 해요:

1. 앱스토어 커넥트 / 구글플레이 콘솔에 인앱상품 등록 (가격, 상품 ID)
2. 클라이언트에서 `purchaseDiamonds` 소켓 emit 대신 IAP SDK 호출
3. 구매 성공 시 받은 영수증(receipt)을 서버로 전달
4. 서버에서 Apple/Google 서버에 **영수증 검증** 요청 (위변조 방지 핵심 단계)
5. 검증 성공 시에만 다이아 지급

지금 구조는 3~5단계가 들어갈 자리(`purchaseDiamonds` 이벤트)를 이미 잡아뒀기 때문에,
나중에 영수증 검증 로직만 끼워넣으면 됩니다. 이 부분은 Claude Code로 이어서 진행하는 걸 추천해요.

## 폴더 구조

```
pokdeng-mp/
├── shared/gameLogic.js     # 서버·클라이언트 공용 룰

│   ├── server.js           # 소켓 서버 (지갑/마켓/로비/테이블 로직)
│   ├── package.json
│   └── public/index.html   # 모바일 앱 셸 클라이언트 (로비/마켓/게임 탭)
```

## 로컬 실행

```bash
cd server
npm install
npm start
```
→ `http://localhost:3001` 접속

## 배포

Render.com / Railway.app에 GitHub 저장소 연결 → Root Directory `server`,
Build Command `npm install`, Start Command `npm start` (이전과 동일).

## 이번에 추가된 게임성 개선 사항

- **베팅/턴 타임아웃**: 베팅 15초, 턴 15초 안에 반응 없으면 자동으로 최소 베팅 / 스테이 처리되어 게임이 멈추지 않아요.
- **무료 칩 받기**: 마켓 화면에서 4시간마다 500칩 무료 충전 (쿨다운 카운트다운 표시).
- **사운드 이펙트**: 외부 음원 파일 없이 Web Audio API로 딜/베팅/승리/팍/패배 효과음을 생성합니다. 상단 🔊 아이콘으로 끄고 켤 수 있어요.
- **잔액 카운트업 애니메이션**: 칩/다이아 숫자가 바뀔 때 스르륵 올라가는 연출.
- **최근 라운드 히스토리**: 테이블 화면 상단에 최근 승/패/무 결과가 점으로 표시돼요 (세션 내에서만 유지).

## 알아둘 점 / 다음 개선 후보

- 지갑은 `socket.id` 기준이라 새로고침하면 초기화돼요. 실제 서비스에서는 로그인(계정) 시스템과 DB(예: Firebase, Supabase, PostgreSQL)로 지갑을 영구 저장해야 합니다.
- 다이아 구매는 위에서 설명한 것처럼 mock 상태입니다.
- 재접속 세션 복구는 아직 없습니다 (새로고침 시 새 플레이어로 인식).
- 다음 후보: 일일 출석 보너스, 업적/미션 시스템, 랭킹, 프로필 커스터마이징, **관전 모드**(꽉 찬 방/진행 중인 방을 구경하다 다음 라운드에 자동 입장), **채팅/이모지 리액션**, 접속 상태(핑) 표시.
